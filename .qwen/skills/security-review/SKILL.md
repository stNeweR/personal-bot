# Security Review

Этот навык предназначен для проведения ревью кода с точки зрения безопасности. Используйте при проверке кода перед релизом, при аудите безопасности или при работе с чувствительными данными.

## Область проверки

При анализе кода выявляю следующие категории уязвимостей:

### 1. Инъекции (Injection)
- **SQL Injection** — конкатенация строк в запросах, отсутствие параметризации
- **Command Injection** — передача пользовательских данных в shell-команды
- **Path Traversal** — доступ к файлам за пределами разрешённой директории

### 2. Межсайтовый скриптинг (XSS)
- Отсутствие экранирования вывода в HTML
- Опасное использование `innerHTML`, `dangerouslySetInnerHTML`
- Недостаточная санитизация пользовательского ввода

### 3. Подделка межсайтовых запросов (CSRF)
- Отсутствие CSRF-токенов в формах
- Использование GET для state-changing операций

### 4. Аутентификация и авторизация
- Слабая валидация сессий
- Недостатки контроля доступа (IDOR, privilege escalation)
- Хранение чувствительных данных в логах

### 5. Криптография
- Использование слабых алгоритмов (MD5, SHA1, DES)
- Неправильная генерация случайных значений
- Хардкод секретов и ключей

### 6. Конфигурация безопасности
- Отладочная информация в production
- Отсутствие security-заголовков
- Чрезмерные CORS-разрешения

## Процесс проверки

1. **Анализ точек входа** — определяю все места получения пользовательских данных
2. **Трассировка потока данных** — отслеживаю путь данных через приложение
3. **Проверка санитизации** — ищу места экранирования/валидации
4. **Оценка рисков** — классифицирую уязвимости по критичности
5. **Рекомендации** — предлагаю конкретные исправления с примерами кода

## Формат отчёта

Для каждой найденной проблемы предоставляю:

```
## [CRITICAL/HIGH/MEDIUM/LOW] Название уязвимости

**Файл:** `path/to/file.php:line`

**Описание:** Краткое описание проблемы

**Риск:** Что может произойти при эксплуатации

**Рекомендация:** Как исправить

**Пример исправления:**
```php
// Было (уязвимо)
$query = "SELECT * FROM users WHERE id = " . $_GET['id'];

// Стало (безопасно)
$stmt = $pdo->prepare("SELECT * FROM users WHERE id = ?");
$stmt->execute([$_GET['id']]);
```
```
